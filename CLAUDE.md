# Claude Code Rules

## 수정 금지 원칙

이 프로젝트의 모든 파일은 **사용자의 명시적 허가 없이 수정할 수 없습니다.**

- 파일을 읽고 분석하는 것은 허용
- 파일을 수정, 삭제, 생성하려면 **반드시 사용자에게 먼저 어떤 파일을 어떻게 바꿀 것인지 설명하고 승인을 받을 것**
- "개선", "리팩토링", "정리" 등의 이유로 요청받지 않은 변경을 하지 말 것
- 버그를 발견하더라도 즉시 수정하지 말고, 사용자에게 보고한 뒤 지시를 기다릴 것

## 현재 안정 버전

- **v1.0.0** (태그: `git tag v1.0.0`)
- 이 버전은 검증 완료된 기준선(baseline)입니다
- 대규모 변경 시 반드시 새 브랜치에서 작업할 것

## 커밋 규칙

- 사용자가 명시적으로 커밋을 요청한 경우에만 커밋할 것
- 커밋 전 변경 내용을 사용자에게 요약해서 보여줄 것
